Hackathon Demonstration#

Beyond the two previous examples, which closely followed the Langchain custom agent tutorial, the goal for the final presentation was to highlight the use of LangSim in production. For this purpose the functionality was transferred from the Jupyter notebook to a python module. As a first step the LangSim module is loaded using the reload_ext jupyter magic:

%reload_ext langsim

Then the Open AI API key is set to the OPENAI_API_KEY environment variable so it can be used by LangSim internally:

import os, getpass
os.environ['OPENAI_API_KEY'] = getpass.getpass("Enter your API key")

Dialog#

From this point on the user can chat with the LangSim interface using the chat jupyter magic. As a first challenge the user communicates their interest to find a material with a bulk modulus in the range of 200 GPa. Following with a clarification of this statement by asking the LLM to calculate the bulk modulus for the noble metals. In contrast to the previous example the LLM is tasked with repeating the same calculation multiple times and then identify the chemical element which most closly matches the targeted bulk modulus of 200GPa. All the required logic is not implemented explicitly, rather the LLM only has access to an get_equilibrium_lattice() function to calculate the equilibrium structure and an get_bulk_modulus() function which calculates the bulk modulus for a given equilibrium structure.

%%chat
I am looking for a material with a bulk modulus in the range of 200 GPa.
Can you compute the bulk modulus for the noble metals for me?

> Entering new AgentExecutor chain...
Sure, I can help with that. The noble metals include gold (Au), silver (Ag), and platinum (Pt). Which model would you like to use for the calculations: EMT or MACE?

> Finished chain.

Sure, I can help with that. The noble metals include gold (Au), silver (Ag), and platinum (Pt). Which model would you like to use for the calculations: EMT or MACE?

The LLM is instructed by the system prompt to clarify which simulation method it should use to calculate a given material property. A the time of the demonstration, two simulation methods were implemented, EMT and MACE. As the user in this demonstration is not aware of the technical details of these simulation methods, he asks for clarification. This highlights how LLMs can use the context of the previous conversation to answer a given question:

%%chat 
What are those methods?

> Entering new AgentExecutor chain...
Certainly! Here is a brief overview of the two models:

1. **Effective Medium Theory (EMT)**:
   - **Description**: EMT is a computationally efficient, analytical model that describes the macroscopic properties of composite materials.
   - **Advantages**: It is relatively fast and can provide good approximations for certain types of materials.
   - **Limitations**: It may not capture all the complexities of atomic interactions, especially for materials with complex bonding.

2. **MACE (Machine Learning Force Field)**:
   - **Description**: MACE is a machine learning force field for predicting many-body atomic interactions that covers the periodic table.
   - **Advantages**: It leverages machine learning to provide highly accurate predictions of atomic interactions and material properties.
   - **Limitations**: It can be computationally more intensive compared to EMT.

Please choose one of these models for the calculations.

> Finished chain.

Based on the explanation the user chooses the EMT, primarily because it is computationally more efficient to calculate a large number of calculation in a short amount of time. The LLM then executes two calculation for each of the three noble metals silver, gold and platinum. These calculations are the calculation of the equilibrium structure get_equilibrium_lattice() which in addition to the chemical symbol now also takes the name of the chosen calculator as an input and the calculation of the bulk modulus get_bulk_modulus() which takes the optimized structure as AtomsDict() object as an input in addition to the chosen calculator as string.

The results are summarized in a short list. This demonstration highlights that LangSim can be used without prior technical experience in calculating material properties. From students to teachers and professors, it is a handy tool for everybody who wants to have quick access to material properties computed with one of the standard methods like EMT. Obviously, the MACE method is more accurate than the EMT method, but also computationally more expensive.

Still to further highlight the real-life application of a tool like LangSim the next step after the completion of those calculation is the comparison to an experimental reference. For this purpose the get_experimental_elastic_property_wikipedia() function is implemented which downloads the experimental values for the elastic properties from the wikipedia site for elastic properties.

Finally, as last step the user tasks the LLM to compare the theoretical predictions and the experimental results to give an indication of the trust worthiness of these calculation. Given the optimization for computational efficiency the result for Platinum with an error of over 20% is insufficient for any real life application. Still this can be addressed by switching to the MACE model.

%%chat
Can you show me the deviation in percent?

> Entering new AgentExecutor chain...
Certainly! Let's calculate the percentage deviation between the calculated and experimental bulk moduli for the noble metals.

The formula for percentage deviation is:
\[ \text{Percentage Deviation} = \left( \frac{\text{Calculated Value} - \text{Experimental Value}}{\text{Experimental Value}} \right) \times 100 \]

Here are the calculations:

1. **Gold (Au)**:
   - Calculated (EMT): 173.84 GPa
   - Experimental: 180.0 GPa
   - Percentage Deviation: \[ \left( \frac{173.84 - 180.0}{180.0} \right) \times 100 \approx -3.42\% \]

Summary#

The jury was convinced that this demonstration of the capabilities of LangSim highlight a future direction in the application of LLMs for materials and chemistry. So the LangSim team won the first price sponsored by RadicalAI.

In comparison to the previous examples, the highlight of this presentation besides hiding the technical complexity by developing a python module was the capability of the LLM to repeat the workflow consisting of two calculation, the get_equilibrium_lattice() function and the get_bulk_modulus() function, over three elements without the need to define any additional functionality.
