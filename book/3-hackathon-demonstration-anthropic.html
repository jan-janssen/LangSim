Hackathon Demonstration#

Beyond the two previous examples, which closely followed the Langchain custom agent tutorial, the goal for the final presentation was to highlight the use of LangSim in production. For this purpose, the functionality was transferred from the Jupyter notebook to a Python module. As a first step the LangSim module is loaded using the %reload_ext jupyter magic command:

%reload_ext langsim

Then the Anthropic API key is set to the LANGSIM_API_KEY environment variable so it can be used by LangSim internally, in addition we set the variables LANGSIM_PROVIDER to select anthropic and LANGSIM_MODEL to select claude-3-5-sonnet-20240620:

import os, getpass
os.environ["LANGSIM_PROVIDER"] = "anthropic" 
os.environ["LANGSIM_API_KEY"] = getpass.getpass("Enter your API key")
os.environ["LANGSIM_MODEL"] = "claude-3-5-sonnet-20240620"

Dialog#

From this point on the user can chat with the LangSim interface using the chat jupyter magic. As a first challenge the user communicates their interest to find a material with a bulk modulus in the range of 200 GPa; following with a clarification of this statement by asking the LLM to calculate the bulk modulus for the noble metals. In contrast to the previous example, the LLM is tasked with repeating the same calculation multiple times and then identify the chemical element, which most closely matches the targeted bulk modulus of 200GPa. None of the required logic is implemented explicitly, rather the LLM only has access to a get_equilibrium_lattice() function to calculate the equilibrium crystal structure and a get_bulk_modulus() function, which calculates the bulk modulus for a given equilibrium crystal structure.

The LLM is instructed by the system prompt to clarify which simulation method it should use to calculate a given material property. At the time of the demonstration, two simulation methods were implemented, EMT and MACE. As the user in this demonstration is not aware of the technical details of these simulation methods, he asks for clarification. This highlights how LLMs can use the context of the previous conversation to answer a given question:

Based on the explanation of the LLM, the user chooses the EMT method, primarily because it is computationally more efficient to calculate a large number of calculations in a short amount of time. The LLM then executes two calculations for each of the three noble metals silver (Ag), gold (Au) and platinum (Pt). These calculations are the calculation of the equilibrium structure get_equilibrium_lattice() which in addition to the chemical symbol now also takes the name of the chosen calculator as an input and the calculation of the bulk modulus get_bulk_modulus() which takes the optimized structure as AtomsDict() object as an input in addition to the chosen calculator as string.

The results are summarized in a short list. This demonstration highlights that LangSim can be used without prior technical experience in calculating material properties. From students to teachers and professors, it is a handy tool for everybody who wants to have quick access to material properties computed with one of the standard methods like EMT. Obviously, the MACE method is more accurate than the EMT method, but also computationally more expensive.

Still, to further highlight the real-life application of a tool like LangSim the next step after the completion of these calculations is the comparison to an experimental reference. For this purpose, the get_experimental_elastic_property_wikipedia() function is implemented which downloads the experimental values for the elastic properties from the wikipedia site for elastic properties.

Summary#

The jury was convinced that this demonstration of the capabilities of LangSim highlights a future direction in the application of LLMs for materials and chemistry. So the LangSim team won the first prize sponsored by RadicalAI.

In comparison to the previous examples, the highlight of this presentation, besides hiding the technical complexity by developing a python module, was the capability of the LLM to repeat the workflow consisting of two calculations, the get_equilibrium_lattice() function and the get_bulk_modulus() function, over three elements without the need to define any additional functionality.
